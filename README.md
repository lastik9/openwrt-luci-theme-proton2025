# openwrt-luci-theme-proton2025

Установщик темы **Proton2025** ([luci-theme-proton2025](https://github.com/ChesterGoodiny/luci-theme-proton2025)) для LuCI на **OpenWrt 25 (apk)**: один интерактивный скрипт, который сам узнаёт с GitHub точное имя свежего `.apk`, скачивает его, ставит, делает тему активной и умеет так же чисто удалять — без ручного ввода команд и подстановки имени файла.

Скрипт `proton2025.sh` — это меню из нескольких пунктов: поставить/обновить тему, посмотреть статус, удалить. Имя `.apk`-файла в релизах меняется от версии к версии и содержит номер сборки, поэтому вписывать его руками неудобно — скрипт находит актуальный файл сам.

### Зачем

Начиная с OpenWrt 24.10 и снапшотов проект перешёл с пакетного менеджера **opkg** (`.ipk`) на **apk** (`.apk`). Тема Proton2025 в официальный фид не входит — её релизы лежат на GitHub. Инструкция автора предлагает ставить `.apk` командой с шаблоном:

```
wget .../releases/latest/download/luci-theme-proton2025-*.apk
apk add --allow-untrusted luci-theme-proton2025-*.apk
```

Проблема в том, что `wget` **не раскрывает `*`** в URL — звёздочка уходит на сервер буквально, файла с таким именем нет, и загрузка падает с **404**. Дальше `apk add` получает несуществующий локальный шаблон и ругается `no such package`. Подразумевается, что вы сами подставите точное имя файла из релиза (например `luci-theme-proton2025-1.1.2-r1.apk`). Скрипт снимает эту рутину: обращается к GitHub, вытаскивает **точную** ссылку на свежий `.apk`, скачивает, ставит через `apk add --allow-untrusted` и переключает LuCI на Proton2025 одной командой.

### Что делает скрипт

При запуске показывает меню `[1-4]`:

1. **Установить / обновить тему** — определяет последний релиз [luci-theme-proton2025](https://github.com/ChesterGoodiny/luci-theme-proton2025), находит точную ссылку на `.apk`, скачивает его в `/tmp`, ставит через `apk add --allow-untrusted`, затем прописывает `luci.main.mediaurlbase='/luci-static/proton2025'` и перезапускает `uhttpd`. Этим же пунктом тема обновляется — всегда тянет свежую версию.
2. **Показать статус** — выводит активную тему (`mediaurlbase`) и то, установлен ли пакет `luci-theme-proton2025` и какой версии.
3. **Удалить тему** — возвращает стандартную тему `bootstrap`, сносит пакет через `apk del`, перезапускает `uhttpd`.
4. **Выход**.

Ссылку на релиз скрипт ищет в два приёма: сначала через **GitHub API** (быстро), а если тот недоступен или упёрся в лимит запросов — через **атом-фид релизов + expanded_assets** (без лимита). Оба способа не требуют `jq`: JSON и HTML разбираются штатными `grep`/`sed`. Работает и с `curl`, и с `wget`.

### Требования

- OpenWrt **25.x** с пакетным менеджером **apk** (для opkg-сборок ставьте `.ipk` по инструкции автора темы).
- **Интернет на роутере** на момент установки — качается пакет темы с GitHub.
- Рабочий **HTTPS** с сертификатами (`ca-bundle` / `ca-certificates`).
- Доступ по **SSH** и права root.

### Установка

Команды выполняются **на роутере** (по SSH):

```
wget https://raw.githubusercontent.com/lastik9/openwrt-luci-theme-proton2025/main/proton2025.sh
sh proton2025.sh
```

В меню выбери пункт **1**. После установки обнови LuCI в браузере с **Ctrl+F5** (иначе останется старый CSS из кэша).

Настройки вынесены в переменные в шапке скрипта: репозиторий темы, имя пакета, путь `mediaurlbase` — правятся в одном месте.

### Удаление

Тот же скрипт, пункт **3**:

```
sh proton2025.sh
```

Возвращает стандартную тему `bootstrap` и удаляет пакет `luci-theme-proton2025`. Безопасен к повторному запуску.

### Известные болячки

- **Звёздочка в команде из README темы** — `wget` не раскрывает `*` в URL, поэтому дословная команда автора падает с 404. Скрипт обходит это, подставляя точное имя файла. Если ставишь вручную — сначала посмотри реальное имя `.apk` на [странице релизов](https://github.com/ChesterGoodiny/luci-theme-proton2025/releases) и подставь его целиком.
- **`.apk` должен быть собран OpenWrt SDK** — валидный пакет для `apk add` делает только официальный SDK/buildroot. Начиная с v1.1.2 автор собирает `.apk` штатно через SDK, поэтому установка на apk-систему рабочая. Старые «переупакованные» `.apk` могли не ставиться.
- **Лимит GitHub API** — без авторизации 60 запросов в час на IP. Если упёрся — скрипт сам переключается на запасной способ (атом-фид), у него лимита нет.
- **Ошибка загрузки / SSL** — если ругается на сертификаты, поставь их: `apk add ca-bundle ca-certificates`.
- **Интерфейс выглядит по-старому** после установки/обновления — это кэш браузера. Обнови страницу с **Ctrl+F5**.
- **opkg-сборки не поддерживаются** — скрипт для apk. На старых сборках (только `opkg`) он сразу сообщит об этом и выйдет.
- **Правишь скрипт на Windows?** Сохраняй в переводах строк **LF (Unix)**. CRLF в `#!/bin/sh` ломает запуск на роутере. Подстраховано файлом `.gitattributes`.

### Диагностика

```
apk info -e luci-theme-proton2025            # установлен ли пакет (код возврата 0 = да)
apk list --installed | grep -i proton2025    # версия установленного пакета
uci get luci.main.mediaurlbase               # активная тема (должно быть /luci-static/proton2025)
ls -l /www/luci-static/proton2025            # файлы темы на месте?
logread | grep -i uhttpd                      # лог веб-сервера LuCI
```

### Проверено на

OpenWrt 25.12.5 (mediatek/filogic, `aarch64_cortex-a53`), тема Proton2025 v1.1.2+.
